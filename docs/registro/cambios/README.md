# Registro de Cambios Importantes

Guardar aquí cambios relevantes realizados en el sistema.

Formato recomendado por archivo:

```md
# YYYY-MM-DD - <titulo>

## Contexto
- Qué problema/objetivo motivó el cambio.

## Cambios aplicados
- Lista concreta de cambios.

## Impacto esperado
- Qué comportamiento cambia (o no cambia).

## Validación
- Tests/checks ejecutados.

## Riesgos y rollback
- Riesgo principal.
- Cómo revertir.
```
