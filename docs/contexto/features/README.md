# Contexto de features generado por PR

Esta carpeta contiene contexto incremental generado automáticamente desde `pull_request`.

Objetivo:

- dejar memoria operativa por feature
- facilitar continuidad entre agentes y revisores
- centralizar contexto funcional, arquitectura tocada, decisiones visibles y notas de UI

Reglas:

- los archivos se generan y actualizan automáticamente desde CI
- cada PR tiene un archivo propio `pr-<numero>-<slug>.md`
- estos documentos complementan, pero no reemplazan, la documentación manual canónica del dominio
